# Умный дом на основе home assistant

## 1. Обзор системы
Проект представляет собой программно-аппаратный комплекс для мониторинга окружающей среды и управления бытовыми устройствами в рамках экосистемы умного дома. Система базируется на центральном сервере **Home Assistant** и исполнительных узлах на базе микроконтроллеров **ESP32/ESP8266**, взаимодействующих по протоколу ESPHome через Wi-Fi соединение.

## 2. Архитектура системы

### 2.1. Программная архитектура
Система строится по трехуровневой модели:
1.  **Уровень управления (Core):** Сервер Home Assistant (Raspberry Pi), выполняющий роль агрегатора данных, графического интерфейса и движка автоматизаций.
2.  **Уровень связи (Transport):** Протокол Native API ESPHome, обеспечивающий двустороннюю связь с минимальными задержками и автоматическое обнаружение устройств.
3.  **Уровень исполнения (Edge):** Прошивка ESPHome на микроконтроллере, обеспечивающая непосредственное чтение датчиков и управление нагрузкой.

### 2.2. Аппаратная схема (Hardware Topology)
**Центральный узел:**
*   **Контроллер:** Raspberry Pi (3B+/4/5).
*   **Среда исполнения:** Home Assistant OS.
*   **Сеть:** Wi-Fi 2.4GHz / Ethernet.

**Периферийный узел (ESP Node):**
*   **Контроллер:** ESP8266
*   **Сенсоры:**
    *   Температура и влажность: BME280 -> подключение к GPIO.
    *   Освещенность:  BH1750.
*   **Исполнительные устройства:**
    *   Освещение: Реле (Digital Output).
    *   Увлажнитель: Реле .
    *   Кондиционер: ИК-передатчик (IR LED) для эмуляции пульта ДУ.

## 3. Конфигурация исполнительного узла (ESPHome YAML)

Ниже приведен эталонный файл конфигурации для прошивки контроллера. Данный конфиг включает поддержку всех датчиков и логику управления.

```yaml
substitutions:
  node_name: smart-climate-node
  friendly_name: "Климат-Контроллер Зал"

esphome:
  name: ${node_name}
  friendly_name: ${friendly_name}

esp32:
  board: esp32dev

# Базовые настройки сети
wifi:
  ssid: "YOUR_WIFI_SSID"
  password: "YOUR_WIFI_PASSWORD"
  ap: {}

api:
  encryption:
    key: "YOUR_ENCRYPTION_KEY"

ota:
  platform: esphome

# Датчики (Sensors)
sensor:
  # Датчик температуры и влажности
  - platform: dht
    pin: 4
    model: DHT22
    temperature:
      name: "Температура в комнате"
      id: room_temp
    humidity:
      name: "Влажность в комнате"
      id: room_humidity
    update_interval: 60s

  # Датчик освещенности (I2C BH1750)
  - platform: bh1750
    name: "Уровень освещенности"
    address: 0x23
    update_interval: 30s

# Исполнительные устройства (Actuators)
switch:
  # Освещение
  - platform: gpio
    pin: 12
    name: "Основное освещение"
    id: light_switch

  # Увлажнитель
  - platform: gpio
    pin: 14
    name: "Увлажнитель воздуха"
    id: humidifier_switch

# Кондиционер (Climate через IR)
ir_transmitter:
  pin: 13
  carrier_duty_percent: 50%

climate:
  - platform: climate_ir
    name: "Кондиционер"
    sensor: room_temp
    # Пример для протокола Samsung (замените на ваш бренд)
    visual:
      min_temperature: 16
      max_temperature: 30
      temperature_step: 1
```

## 4. Регламент развертывания программной части

Установка системы производится в четыре этапа: подготовка сервера, настройка узла, прошивка и интеграция.

### Этап 1: Подготовка Raspberry Pi
1.  Скачайте образ **Home Assistant OS** с официального сайта.
2.  Запишите образ на SD-карту (мин. 32ГБ) с помощью утилиты **BalenaEtcher**.
3.  Вставьте карту в Raspberry Pi, подключите питание и интернет.
4.  Дождитесь инициализации по адресу `http://homeassistant.local:8123`.

### Этап 2: Установка аддона ESPHome
1.  В интерфейсе Home Assistant перейдите в **Settings** (Настройки) -> **Add-ons** (Дополнения).
2.  Нажмите **Add-on Store**, найдите **ESPHome** и нажмите **Install**.
3.  После установки активируйте "Show in sidebar" и нажмите **Start**.

### Этап 3: Создание и прошивка конфигурации
1.  Откройте панель ESPHome в боковом меню.
2.  Нажмите **+ New Device**, введите имя `smart-climate-node`.
3.  Выберите тип процессора (ESP32).
4.  Скопируйте содержимое YAML-конфигурации из раздела 3 данного документа в редактор ESPHome (заменив стандартный код).
5.  Подключите ESP32 к Raspberry Pi по USB.
6.  Нажмите **Install** -> **Plug into this computer** (или выбор USB порта на малинке).
7.  Дождитесь завершения компиляции и процесса "Flash".

### Этап 4: Финальная интеграция в Home Assistant
1.  После успешной прошивки перейдите в **Settings** -> **Devices & Services**.
2.  Home Assistant автоматически обнаружит новое устройство через протокол API.
3.  Нажмите **Configure** и введите ключ шифрования (API Key), указанный в конфиге.
4.  Все объекты (датчики, переключатели, климат) появятся в системе автоматически и будут доступны для создания автоматизаций и вывода на Dashboard.
